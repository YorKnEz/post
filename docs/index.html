Introduction

Purpose

Realize a Web application for translating poems commented by multiple users. Translation for the poem or a fragment of it may be provided with the afferent annotation. The system will support lyric annotation with the possibility of sharing on social platforms like Tumblr or WordPress, based on the public APIs. Statistics will be provided regarding the activity on the platform (e.g. the most active translator, the most annotated poem, ...). The new translations will be exposed on an RSS flux. The essential functionalities will be provided by our own API, built based on the architectural styles of REST or GraphQL.

Intended Audience and Reading Suggestions

This document is intended for those that want to take a look at our source code. This document provides detailed documentation for our product and is required to read before taking a look at the source code. The document is intended to be read in the given order.

Product Scope

Product that aims to provide internationalization of poems by the community and ease of sharing knowledge about said poems.

Product Functions

Regular user functions

  • Create an account
  • Log into an account
  • Reset password
  • See the latest lyrics/translations
  • See global statistics
  • See user profiles
  • See lyrics/annotations
  • Add lyrics/annotations
  • Edit lyrics/annotations
  • Share lyrics/annotations on social media
  • Change the language on the lyrics page
  • Become a poet

Poet user (new) functions

  • Can edit without requesting permission any of his lyrics and annotations on his lyrics

Admin user (new) functions

  • Approve/deny poets on the app
  • Approve/deny new lyrics/annotations
  • Approve/deny new edits of lyrics/annotations

User Classes and Characteristics

There will be three types of users which were already sketched in the Product functions section:

  • Regular
  • Poet
  • Admin

Regular

The regular user is the average user that uses the application. He can contribute to lyrics by editing them and annotationg them, but he requires the approval of an admin.

Poet

The poet is a special kind of user that has the abillity to do whatever he wants without requesting permission to any piece of content that is his own.

Admin

The admin is the one that approves the requests of the regular users and, in general, handles moderation of the application. He can approve poets on the app, new poems, new annotations, edits, etc.

Operating Environment

The app has been extensively tested only on Firefox browser. It may be used on other browsers as well but we do not guarantee 100% compatibility.

Design and Implementation Constraints

The main limitation of this product is the fact that we are not allowed to use any frameworks for this project, everything must be done from scratch. At the same time, there is no guarantee that the pages of the app are 100% compatbile with other browsers than Firefox.

User Documentation

We do not provide any kind of user documentation. Our application is designed to be easy to use and intuitive and, as such, we do not see the need for a user manual or any other document of that type.

Assumptions and Dependencies

We will use APIs of social media apps in order to provide the share feature.
